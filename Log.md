### c21 永馨復健科 API 抓取方式（2026-02-21）

**問題**：source_url 填的是 `sc-dr.com.tw`（HIS 廠商），不是診所預約頁面的正確 URL。

**正確 URL**：`https://www.sc-dr.com.tw/progress/3531035027.php`

**抓取方法**：頁面用 AJAX 動態載入，直接 POST 呼叫 API：

```bash
curl -s -X POST \
  -d "hospid=3531035027&firstday=115.02.21&lastday=115.03.20" \
  "https://www.sc-dr.com.tw/progress/ajax/getreserve.php"
```

- `firstday` / `lastday` 格式為**民國年**（例：115.02.21 = 2026-02-21）
- `infirm1x` = 上午，`infirm2x` = 下午，`infirm3x` = 晚上
- 診所時間：09:00-12:00 / 14:30-17:30 / 18:00-21:00（由 `load.php` 的 `lnp_time` 欄位確認）
- 醫師欄格式：`姓名      號碼`（需去除尾部空白+數字）

**醫師**：陳彥誌（固定）、陳瑋杰（週四午/週五晚）、王欣宜（週一早）

### CXMS 班表抓取方式確認（2026-02-21 更正）

**舊認知（錯誤）**：~~AJAX 動態載入，靜態爬蟲抓不到，需要 Selenium~~

**正確做法**：用 `curl` 以 **HTTP**（非 HTTPS）直接 GET 即可，班表資料完整嵌在靜態 HTML 中：

```bash
curl -s -L --max-time 10 "http://web.cxms.com.tw/{code}/hosp.php"
```

**根本原因分析**：
- 頁面 JS 確實有 AJAX（`makeRequest("docsch_ajax.php")`），但作用是每 2 分鐘刷新，不是首次載入
- 首次載入時班表資料已完整嵌在靜態 HTML 的 `<table>` 中
- 之前嘗試失敗原因：WebFetch 工具走 HTTPS（`web.cxms.com.tw` 的 HTTPS 拒絕連線）

**已於 2026-02-21 補齊** c02/c03/c04/c05/c06/c07/c20 週四五班表（共 46 筆）
**SOP 已更新**：v1.2 → v1.3，類型 A 操作步驟改為 curl + Python 解析，不再需要截圖 + OCR

## 功能 Bug 修正

### 設定頁 Toggle 三態邏輯 Bug

**問題**：醫師顯示/隱藏 toggle 第二次點擊行為錯誤。

**解法**：三態邏輯：
1. 無 override → 點擊 → 寫入 `'hide'`
2. 有 `'hide'` → 點擊 → 刪除 key（回到基礎狀態）
3. 基礎狀態本來就隱藏 → 點擊 → 寫入 `'show'`
